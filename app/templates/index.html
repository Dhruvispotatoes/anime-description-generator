The Process

Step 1: Cleaning the data

The first step in AI machine learning is preprocessing, or "cleaning up" data. We found a dataset on Kaggle with over 14000 anime scraped from Anime-Planet, but after a few preprocessing steps with the Python pandas library, we ended up with around 8000 usable descriptions.

Step 2: Training the model

In order to generate text, we chose to fine-tune a GPT-2 model with aitextgen. The model we chose had 124M parameters, and after a few rounds of trial and error, the model was able to generate coherent descriptions. The training time for the final model was about 35 minutes.

Step 3: Creating the interface

We created this page and the try-it interface for our model using HTML, CSS, and JavaScript, and we're using Flask to serve pages and send the model's output. Our form takes a prompt, the number of descriptions to generate, and the temperature of the model, generating an output accordingly.

Challenges We Faced

The main challenge we faced was actually the limitation of our original idea. The idea our team had originally planned on was to generate descriptions based on a given title, but training the model to do this didn't work out. The titles were often in Romaji, so the model would be confused by the non-English text often at the beginning of its training data. Additionally, a title doesn't give enough information for a consistent description anyways, so we decided to change the model to take the start of a description and output the rest.

We also had a hard time managing our computing resources. To collaborate, we used CoCalc servers, but the server couldn't handle much training and ended up crashing several times when a few of us were testing.

Of course, there were a lot of minor bugs that we had to squash along the way, but the experience taught us a lot. We hope to use the knowledge we gained from this project in the future.
